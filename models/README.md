# ESP-VISION Models

## Overview

This directory stores optional model files for ESP-VISION examples.

Models are not built into firmware.

## Usage

Copy the required `.espdl` files to board storage, such as `/sdcard` or `/flash`.

Keep the directory layout under `models/` when possible.

## Metadata sidecar

Models may live in any subfolder of `models/`. Every model file ships with a same-named `*.json` sidecar describing it (for example `espdet_pico_224_224_cat.espdl` and `espdet_pico_224_224_cat.json`); the model binary is the same-stem sibling and may use any extension (`.espdl` today, other formats later). The website model list (config.toml `[[website.models]]`, generated by `tools/ci/launchpad/generate_launchpad.py`) is built by scanning all of `models/` for these sidecars, so a model in any subfolder is picked up as soon as it has a valid sidecar — adding one is just dropping in both files, no code change. A sidecar is recognized by its `architecture` and `api` keys; any other JSON under `models/` is ignored.

Required keys:

| Key | Meaning |
|-----|---------|
| `name` | Display name, e.g. `ESPDet Pico Cat` |
| `architecture` | Model architecture, e.g. `ESPDet Pico`, `YOLO11n` |
| `api` | Runtime API class, e.g. `espdl.ESPDet` |
| `task` | `detection`, `pose`, ... |
| `input` | Input resolution, e.g. `224x224` |
| `inputFormat` | Pixel format, e.g. `RGB565` |
| `dataset` | Training dataset, e.g. `COCO` |
| `labels` | List of detected labels (empty for large class sets like COCO) |
| `sizeBytes` | Model file size in bytes; cross-checked against the `.espdl` |

Optional keys: `description` (what the model does — do not repeat the resolution/format already captured by other keys), `datasetUrl`.

`file`, `path`, `slug`, and `downloadUrl` are derived from the file location and must not be hand-written in the sidecar. `downloadUrl` points to the same model file on the public `espressif/esp-vision` GitHub `master` branch for the website download button.
